# フォーム→PDF→下書きメール

フォームで集めた回答CSVから、1件ずつ**PDFレポート**と**宛先入りのメール下書き（.eml）**を作る無料ツールです。

**公開URL: https://form-to-pdf-mail.vercel.app/** （インストール不要・登録不要・無料）

## こんな「面倒」をなくします

アンケートや申込フォームの回答者一人ひとりに、お礼とレポートを送りたい——
でも、**1件ずつPDFを作って、メールに宛先と本文を書いて、添付して**…を人数分くり返すのは苦痛です。

このツールは、その**1件あたり約10分の手作業を約30秒**にします（同種業務での想定値であり、効果を保証するものではありません）。

## 使い方（5ステップ）

1. **回答CSVを入れる** — フォームから書き出したCSVを、点線の枠にドラッグ&ドロップ（またはクリック／Enterキーで選択）
2. **列の対応を確認する** — 「宛先メール」「氏名」がどの列かを自動で推測します。**実際の値がサンプル表示される**ので、取り違えていないか必ず目で確かめてください（ここが済むまで次へ進めません）
3. **相手を1人選ぶ** — 回答一覧から送る相手を選びます
4. **レポートを確認する** — 様式を「汎用／診断風／シンプル」から選ぶと、その場でプレビューが切り替わります。**ここに見えているものが、そのままPDFになります**
5. **書き出す** — PDFをダウンロードし、お使いのメールに合わせてどちらかを選びます
   - **A. Outlook・Thunderbird・Apple Mail の方** → 「メール下書き（.eml）をダウンロード」。開くと宛先・件名・本文・PDF添付が入った**下書き**として開きます
   - **B. Gmail など Web版メールの方** → 「本文をコピー」。メール作成画面に貼り付け、画面に表示された宛先を入れ、ダウンロードしたPDFを添付してください

## 安心してお使いいただくために

- **回答データはお使いのパソコンの外に出ません。** すべての処理はブラウザの中だけで行われ、サーバーに送信も保存もされません（実行時の外部通信はゼロです）
- **メールを自動送信することはありません。** このツールが作るのは「下書き」までです。送信は必ずご自身の手で行ってください
- **元のCSVファイルは書き換わりません。** 読み取りだけを行います
- ページを閉じる・再読み込みすると、読み込んだデータは消えます（どこにも残りません）

## 3つの様式

| 様式 | 中身 | 向いている用途 |
|---|---|---|
| **汎用** | ご回答の全項目を表で並べた標準レポート | まずはこれ。アンケートのお礼・控え |
| **診断風** | クリーム地に明朝で組んだ、所見票ふうの落ち着いた誌面 | 診断結果・カルテ・報告書の体裁で渡したいとき |
| **シンプル** | 宛名と回答項目だけの軽い1枚 | 受付控え・簡単な確認書 |

※ 診断風は、いただいた回答をそのまま所見欄に転記します。**AIが勝手に点数や講評を作ることはありません。**

## 対応形式・環境

| 項目 | 内容 |
|---|---|
| 入力 | .csv（1行目=列名、2行目以降=回答。文字コードは UTF-8 / Shift_JIS を自動判別。上限20MB） |
| 出力 | PDFレポート（1件1ファイル）／メール下書き `.eml`／本文テキスト（コピー） |
| 環境 | Windows / Mac の Chrome・Edge（インストール不要・アカウント登録不要） |
| 規模の目安 | 数千件のCSVでも読み込めます（生成は1件ずつ） |
| 操作 | マウスでもキーボードだけでも全ステップ操作できます |

## よくあるつまずき

- **「宛先メールの列を自動で見つけられませんでした」と出る** → STEP 2の「宛先メール」で、メールアドレスが入っている列をご自身で選んでください
- **「メールアドレスの形になっていない値が N件あります」と出る** → 列を選び間違えている可能性が高いです。セレクトの下に出る**実際の値のサンプル**を見て、正しい列を選び直してください
- **列の対応を直したのに反映されない** → 変更しただけでは反映されません。**「この対応で進む」を押す**と、プレビューと下書きが新しい対応で作り直されます（未確定のときは見出しに「! 未確定」と出ます）
- **`.eml` がGmailで開けない** → GmailのWeb版は `.eml` を直接開けません。**「本文をコピー」**の方をお使いください（宛先は画面に表示されます）
- **「1行目の列名より値の数が多くなっています」と出る** → CSVの列名が足りていません。あふれた値はレポートに載らないため、元のCSVをご確認ください
- **PDFの文字化けが心配** → 日本語はそのまま出力されます。プレビューに見えているとおりにPDFになります

## 開発者向け

```bash
npm install
npm run dev      # 開発サーバー
npm run build    # dist/ に出力（静的ホスティングにそのまま置けます）
```

- 構成: Vite + TypeScript（フレームワークなし）／papaparse（CSV解析）／html2pdf.js（PDF生成）
- **依存は npm で同梱し、CDNを使いません**（実行時の外部通信ゼロを実体として保証するため）
- `.eml` は手組みのMIME（multipart/mixed・`X-Unsent: 1`・RFC2047件名・RFC2231添付ファイル名）
- **CSV由来の値は必ず `textContent` 経由で挿入します**（`innerHTML` に生値を連結しない＝XSS防止）。変更時はこの規律を必ず守ってください
- 検収記録: `../acceptance-phase4-20260713.md`（Phase 4〜5-6・G2独立検収まで）
